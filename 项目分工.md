## 五、共同交付标准

每完成一个页面或链路必须检查：

* [ ] 路由可直接访问，且工作台/侧边栏入口可进入
* [ ] 返回、取消、提交、保存、通过、驳回等关键按钮有明确行为
* [ ] 跨角色状态文案一致，下一处理人清楚
* [ ] mock 数据位置清晰，不在多个页面重复维护同一份状态
* [ ] 页面无明显溢出、重叠和空白断层
* [ ] 控制台无明显红错
* [ ] 总控进度表同步更新“第二轮验收状态”和备注
