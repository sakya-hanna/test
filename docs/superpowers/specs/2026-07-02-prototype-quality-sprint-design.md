# 原型质量收敛设计

## 背景与目标

本仓库是无锡学院实习管理系统的 Vue 交互原型，不承担真实后端、权限和持久化实现。本轮在三天内按问题类别统一优化全仓库，不按成员或角色拆分所有权。

目标是让原型具备稳定展示、流程可达和清晰交接能力：页面视觉完整且不明显偏移、导航不静默、P0/P1 页面不保留显眼占位、关键展示路径可以从入口走完并返回，同时构建与浏览器运行不报错。

## 约束

- 遵守 `agent.md`：路由闭环、入口可见、非一级页面有返回路径。
- 以现有 Figma、路由规划和进度文档为业务依据，不擅自补充未确认规则。
- 保留 Vue 3 `<script setup>` 与现有 Vue Router、Tailwind CSS 技术栈。
- 复用现有 Layout 和页面模式，不在三天冲刺中进行无关架构重写。
- 原型不是功能系统，不为展示之外的需要补复杂业务状态、跨页面同步或持久化。

## 方案选择

采用“分类横扫 + 核心流程优先”的混合方案。

纯逐页优化容易在少数页面消耗全部时间；纯问题扫描又可能修完按钮外观却未形成业务闭环。混合方案先统一交互和页面完成标准，再用学生申请、审核、周报、评价等关键链路验证结果。

## 《问题汇总》驱动的优先级

最新 `问题汇总.md` 作为本轮第一批验收输入：

1. 登录页内容组在常用 PC 视口保持水平居中。
2. 企业端“企业指导老师”从静默链接改为可到达的纯展示页面；页面编号与业务字段仍标记待确认，不擅自定义流程。
3. 企业注册链接先进入现有 `/portal` 背景介绍展示页，再进入账号注册和四步注册向导。
4. 企业发布岗位表单移除“实习类型”，因为该分类属于学校要求，不是企业发布要求。
5. 企业岗位已发布栏位的对齐已由最新 `main` 修复，本轮只做回归检查。

## 设计

### 1. 登录与布局

PC 登录页不再直接使用 Figma 画布的固定 `left/top` 坐标。Logo、标题和表单组成一个有限宽度的垂直内容组，由外层容器负责水平居中；垂直位置保留接近 Figma 的视觉重心，同时对较矮视口允许滚动或压缩间距。

### 2. 跳转与轻量响应

按钮分为导航按钮和展示按钮两类：

- 导航动作进入已注册路由。
- 展示动作只需打开弹层、切换标签、显示示例提示或明确标注“原型展示，待后端接入”，不要求实现真实业务结果。
- 纯装饰元素不伪装成可点击控件；可点击控件不能静默。

所有响应保持轻量、可观察，不引入新的 UI 框架，也不为了模拟功能建立复杂状态管理。

### 3. 占位页面补全

按 `docs/页面分类优化进度.md` 处理明显占位页。页面内容优先复用对应角色 Layout，并依据 Figma 和已有同类页面构建标题区、统计/筛选区、数据主体和空态。没有业务依据的字段或流转保持“待确认”，不编造规则。

学院端的工作台、企业初审、通知发布为 P0；系统管理端页面为 P1，采用统一后台信息架构补齐可验收主体。

### 4. 展示路径

关键展示路径使用现有路由串联：

- 学生：岗位 → 申请 → 实习列表 → 实习详情 → 周报/月报/材料。
- 审核：教师初审 → 学院复核 → 学校/企业相关状态展示。
- 成绩：企业评价 → 教师评定 → 学生查看。

各页面使用一致的静态演示数据和状态文案；不要求前一页操作改变后一页数据，只要求路径、页面关系和视觉叙事成立。

### 5. 错误与空状态

列表为空时展示空态说明和下一步入口。不存在的 URL 由 404 页面承接，并提供返回入口。原型无法完成的动作使用轻量提示说明，不产生控制台错误。

### 6. 验证

- 契约测试：检测登录页不再依赖固定横向坐标、导航入口不使用空目标、显眼占位文案清零。
- 路由验收：每个已注册路由可以渲染，关键入口和返回路径存在。
- 人工验收：按关键流程逐步点击并检查控制台。
- 最终执行 `npm test` 与 `npm run build`。

## 非目标

- 不实现真实 API、数据库、身份认证、文件上传或权限控制。
- 不实现完整表单校验、审核状态机、跨页面数据同步或持久化。
- 不重新设计所有 Figma 页面。
- 不为未确认业务创建新的流程规则。
